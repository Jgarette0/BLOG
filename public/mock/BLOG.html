Design Patterns are the solutions to commonly occurring problems in software design. These patterns are easily re-usable and are expressive.

According to Wikipedia

In software engineering, a software design pattern is a general reusable solution to a commonly occurring problem within a given context in software design. It is not a finished design that can be transformed directly into source or machine code. It is a description or template for how to solve a problem that can be used in many different situations.

Types of Design Patterns -Creational -Structural -Behavioral

Creational Design Patterns Creational Design Patterns primarily focus on object creation mechanisms, providing flexibility in creating objects and instances. They abstract the instantiation process, making the system independent of how its objects are created, composed, and represented. There are several types of Creational Design Patterns:

1. Singleton Pattern

Ensures a class has only one instance and provides a global point of access to it. Often used for managing resources or configuration settings.

2. Factory Method Pattern

Defines an interface for creating an object but allows subclasses to alter the type of objects that will be created. It provides a way for delegating the instantiation logic to child classes.

3. Abstract Factory Pattern

Offers an interface for creating families of related or dependent objects without specifying their concrete classes. Encourages the use of interfaces for creating instances of related objects.

4. Builder Pattern

Facilitates the construction of complex objects step by step, allowing the same construction process to create different representations. Useful when dealing with objects that have many possible configurations.

5. Prototype Pattern

Creates new objects by copying an existing object, known as the prototype, during runtime. Avoids the complexity of creating objects from scratch by copying existing instances.

6. Object Pool Pattern

Manages a pool of objects, allowing them to be reused rather than creating and destroying them as needed. Enhances performance by recycling objects and minimizing the overhead of creating new ones. These patterns not only emphasize the flexibility and decoupling of the creation process but also contribute to maintaining a codebase that's easier to maintain, extend, and modify. Each pattern addresses specific concerns related to object creation, providing various approaches based on the requirements of the system.
